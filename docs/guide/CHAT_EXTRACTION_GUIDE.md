## ❓ ทำไมต้องใช้ Scraper แทนการดึงผ่าน API?

แม้ Facebook จะมี **Graph API** ให้ใช้ แต่เราจำเป็นต้องใช้ **Scraper** ด้วยเหตุผลสำคัญดังนี้:

1.  **การระบุตัวตนพนักงาน (Admin Attribution):** API ของ Facebook จะบอกเพียงว่า "Page" เป็นคนตอบ แต่จะไม่ระบุชื่อพนักงาน (แอดมิน) รายบุคคล ข้อมูลนี้ปรากฏอยู่เฉพาะบนหน้าจอ Business Suite (ในรูปแบบ "ส่งโดย [ชื่อ]") ซึ่งจำเป็นต่อการคิดค่าคอมมิชชั่นและ KPI
2.  **ข้อจำกัดด้านความเป็นส่วนตัว (Privacy):** ข้อมูลบางอย่าง เช่น ชื่อเล่นแอดมิน หรือสถานะการอ่าน (Read Receipt) ของพนักงาน ไม่ถูกเปิดเผยผ่าน API สาธารณะ
3.  **ความสมบูรณ์ของประวัติ (Full Context):** API มักจำกัดจำนวนข้อความย้อนหลัง (Rate Limit) แต่ Scraper สามารถดึงข้อมูลได้ลึกเท่าที่ตาคนเห็นบนหน้าจอ

## 🛠️ ขั้นตอนที่ 1: เปิด Chrome ในโหมด Remote Debugging (Dev Mode)
เพื่อให้สคริปต์สามารถเข้าไปควบคุมเบราเซอร์ที่เราใช้งานอยู่ได้ เราต้องเปิด Chrome ด้วยคำสั่งพิเศษที่เปิดพอร์ตสื่อสาร (Port 9222) ไว้ครับ

1.  **ปิด Chrome ทั้งหมดก่อน:** (รวมถึงหน้าต่างปกติด้วย)
2.  **รันคำสั่งเปิดหมวด Dev Mode:**
    *   **สำหรับ Mac:** ดับเบิ้ลคลิกไฟล์ `เปิด_Chrome_CRM.command` ในโฟลเดอร์ `automation`

        ```bash
        /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir="$HOME/.chrome-vschool-crm" "https://business.facebook.com/latest/inbox/all"
        ```
3.  **Login Facebook:** ล็อคอินและกดเข้าไปที่เมนู **Inbox** ของ Business Suite ให้เรียบร้อย

## 🚀 ขั้นตอนที่ 2: รันสคริปต์ดึงข้อมูล (Deep Scrape V2)
ใช้ `sync_agents_v2.js` ซึ่งเป็นเวอร์ชันที่ "ขุด" ข้อมูลได้ลึกและแม่นยำที่สุด

### **A. ดึงจากหน้าจอที่เห็น (Sidebar Mode)**
เหมาะสำหรับดึงแชทล่าสุด หรือแชทที่เพิ่งคุยเสร็จ
```bash
node automation/sync_agents_v2.js --attach --limit=50
```
*   `--attach`: เชื่อมต่อเข้ากับ Chrome ที่เปิดไว้แล้ว
*   `--limit=50`: จำนวนแชทที่จะให้ไล่ดึง (จากบนลงล่าง)

### **B. ดึงย้อนหลังตามวันที่ (DB Mode)**
เหมาะสำหรับกู้ข้อมูลเก่า (เช่น เดือนกุมภาพันธ์) โดยดึงรายชื่อแชทจากฐานข้อมูลแล้วไปไล่หาใน Facebook
```bash
node automation/sync_agents_v2.js --attach --mode=db --from=2026-02-01 --to=2026-02-28
```
*   `--mode=db`: ดำน้ำหาตาม List ในฐานข้อมูล
*   `--from / --to`: ระบุช่วงวันที่ต้องการกู้

### **C. บังคับดึงซ้ำ (Force Mode)**
หากต้องการอัปเดตข้อมูลทับของเดิม (ถ้าชื่อคนตอบเปลี่ยนหรือข้อมูลไม่ครบ)
```bash
node automation/sync_agents_v2.js --attach --force
```

## 🧩 สรุปกลไกการทำงาน (Hybrid Logic)

เพื่อให้เข้าใจง่าย ระบบทำงานประสานกัน 3 ส่วนดังนี้:

1.  **API (จดจำ ID):** เราดึงข้อความจาก **Facebook Graph API** มาลงฐานข้อมูลก่อน ซึ่งในขั้นตอนนี้เราจะได้ไอดี **MID (Message ID)** ที่ถูกต้องแม่นยำทุกข้อความ แต่ "ไม่มีชื่อคนตอบ" (เห็นแค่ Page)
2.  **Scraper (จดจำคนตอบ):** เราใช้หุ่นยนต์ (Scraper) เข้าไปอ่านหน้าจอ Business Suite เพื่อดู Label **"ส่งโดย [ชื่อพนักงาน]"** แต่เนื่องจากโครงสร้างเว็บของ Facebook ซับซ้อนมาก หุ่นยนต์จึงมักจะดึงไอดี MID ออกมาตรงๆ ไม่ได้
3.  **Text Matching (ตัวเชื่อม):** เพื่อให้ "ชื่อพนักงาน" (จากข้อ 2) ไปจับคู่กับ "ไอดี MID" (จากข้อ 1) ได้อย่างถูกต้อง เราจึงใช้ **เนื้อหาข้อความ** มาเปรียบเทียบกัน เมื่อข้อความตรงกัน ระบบจะทำการบันทึกชื่อพนักงานคนนั้นลงในไอดี MID ในฐานข้อมูลของเราครับ
