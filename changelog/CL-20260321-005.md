## Summary

แก้ไข Monthly Message Trend chart ใน Admin Performance Dashboard ที่ไม่แสดงข้อมูล เนื่องจาก employee filter ใน API route แคบเกินไป (ดึงเฉพาะ TVS-MKT-* หรือ developer department) ทำให้ Fafah + Aoi (AGENT role) ไม่ถูก include

## Root Cause

`GET /api/analytics/admin-performance` query employees ด้วย:
```javascript
OR: [
    { employeeId: { startsWith: 'TVS-MKT-' } },
    { employeeId: { startsWith: 'TVS-EMP-' }, department: 'developer' },
]
```

Fafah (TVS-EMP-2026-0003, AGENT) และ Aoi (TVS-EMP-2026-0004, AGENT) ไม่ match filter นี้ → `data = []` → `allMonths = []` → กราฟไม่แสดง

## Changes

**`src/app/api/analytics/admin-performance/route.js`:**
- เปลี่ยนจาก hardcoded department filter → query `DISTINCT responder_id` จากตาราง `messages` ก่อน
- ใช้ responder IDs จริง เป็น filter สำหรับ employee query
- ทำให้ทุกพนักงานที่ตอบแชทจริงแสดงในกราฟ ไม่ขึ้นกับ department/role
