### New: `src/mcp/vschool-mcp-server.js`
- stdio transport entry point สำหรับ Claude Desktop

### New: `src/app/api/mcp/route.js`
- Vercel-compatible stateless JSON-RPC endpoint
- `GET /api/mcp` → health check (public, no auth)
- `POST /api/mcp` → JSON-RPC dispatch (`tools/list`, `tools/call`, `initialize`)
- Bearer token auth guard: ถ้าตั้ง `MCP_SECRET` env var จะ require `Authorization: Bearer <secret>`

### Fix: `src/middleware.js`
- เพิ่ม `{ prefix: '/api/mcp', role: null }` ก่อน catch-all `/api` rule
- เดิม `/api/mcp` ตกไปที่ catch-all → require AGENT session → 401 สำหรับ MCP clients ที่ไม่มี cookie
- `/api/mcp` มี Bearer token auth ของตัวเอง → middleware ต้อง bypass session guard

### Updated: `package.json`
- เพิ่ม `"mcp:start": "npx tsx --tsconfig tsconfig.json src/mcp/vschool-mcp-server.js"`
- ใช้ `--tsconfig` flag เพื่อ resolve `@/` path aliases ใน standalone Node.js

### Config: Claude Desktop

- ใช้ `npx tsx --tsconfig` เพื่อ run vschool-mcp-server.js ใน stdio mode

## Root Cause (middleware 401)

`/api/mcp` ตกไปที่ catch-all rule `{ prefix: '/api', role: 'AGENT' }` → middleware เรียก `getToken()` ซึ่งหา NextAuth cookie ใน request — MCP clients ไม่มี cookie จึงได้ 401 แทนที่จะให้ Bearer auth ใน route handler จัดการ

**Fix:** เพิ่ม whitelist rule ก่อน catch-all (first-match wins logic)

## Known Gotchas

- **MCP_SECRET**: ถ้าตั้งใน Vercel → POST ต้องมี `Authorization: Bearer <secret>` ทุก request
- **module boundary**: ห้ามสร้าง `src/mcp/package.json` ที่มี `"type":"module"` — จะ break named exports จาก `src/lib/*.js` (CJS)
- **tsconfig flag**: `npx tsx --tsconfig tsconfig.json` บังคับ — ถ้าไม่มีจะ fail ด้วย `Cannot find module '@/lib/db'`
