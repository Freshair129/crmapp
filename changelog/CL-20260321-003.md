## Breaking Changes

1. **DB role values** → normalize UPPERCASE (`Developer` → `DEVELOPER`)
2. **Force re-login** → เปลี่ยน `NEXTAUTH_SECRET` ใน Vercel เพื่อ expire sessions เดิม
3. **New roles** → MARKETING, HEAD_CHEF ต้องสร้าง employee record ก่อนใช้งาน

---

## Known Gotchas

- `adsOptimizeRepo.js` ใช้ `FB_ACCESS_TOKEN` env var — ต้อง set ใน Vercel Dashboard
- `FB_AD_ACCOUNT_ID` ต้องระบุสำหรับ duplicate campaign feature  
- Lifetime budget approval ต้องมี MANAGER approve ก่อน Meta API จึงจะถูก call
- Permission Matrix UI เป็น read-only ใน v1.4.0 — editable toggles จะมาใน phase ถัดไป
