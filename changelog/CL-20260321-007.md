## Summary

เปลี่ยนรูปแบบ Employee ID จาก `TVS-EMP-[YYYY]-[XXXX]` เป็น `TVS-[TYPE]-[DEPT]-[NNN]` เพื่อแยกประเภทการจ้าง (EMP/FL/CT) และแผนก (MKT/MGT/PD/SLS/AM/ADM/GD/CG/MM/MGFX/ED/CC) ใน ID

## Changes

- **`src/app/api/employees/route.js`** — เพิ่ม `EMPLOYMENT_TYPE` map + อัปเดต `DEPT_CODE` ให้ครอบคลุม 12 แผนก + `generateEmployeeId()` รับ `employmentType` param
- **`src/components/EmployeeManagement.js`** — เพิ่ม dropdown "ประเภทการจ้าง" (EMP/FL/CT) + dropdown "แผนก/ตำแหน่ง" (12 ตัวเลือก) แทน text input
- **`id_standards.yaml`** — อัปเดต Employee ID format spec เป็น v3
- **`src/app/api/analytics/admin-performance/route.js`** — SQL LIKE ครอบคลุม TVS-EMP/FL/CT
- **`src/app/auth/signin/page.tsx`** — placeholder อัปเดตเป็น TVS-EMP-MKT-001
- **`src/lib/authOptions.js`** — comment update

## Breaking Change

- พนักงานใหม่จะได้ ID format ใหม่ (`TVS-EMP-MKT-001`)
- พนักงานเดิม (TVS-EMP-2026-0001~0004) ยังคง ID เก่า ไม่ migrate
