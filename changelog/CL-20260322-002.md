## Summary

รวม ID generators จาก 14 ไฟล์ เข้า `src/lib/idGenerators.js` ไฟล์เดียว + เพิ่ม Agent ID/Code fields ใน Employee + Customer ID เปลี่ยนเป็น YYMM

## Changes

### Centralized ID Generators
- **`src/lib/idGenerators.js`** — NEW: 20 generators, 2 helper patterns (`nextSerialByPrefix`, `nextSerialByCount`)
- **ลบ** `src/utils/idGenerator.js` + `src/lib/id-generators.js`
- **อัปเดต import** ใน 16 ไฟล์ → `@/lib/idGenerators`
- ลบ inline generator functions จาก: kitchenRepo, enrollmentRepo, scheduleRepo, packageRepo, recipeRepo, assetRepo, certificateRepo, employees/route, tasks/route, notifications/rules/route, authOptions

### Agent ID + Agent Code (Prisma schema)
- `Employee.agentId` — `AGT-[TYPE]-[YYMM]-[NNN]` (unique, auto-generated) TYPE: HM=Human, AI=AI
- `Employee.agentCode` — 3-4 uppercase letters IATA-style (unique, manual input, required)
- EmployeeManagement UI: เพิ่ม Agent Code input (auto-uppercase, A-Z only) ใน Add + Edit modal
- Detail panel แสดง Agent ID + Agent Code

### Customer ID Format
- เปลี่ยนจาก `TVS-CUS-[CH]-[YY]-[XXXX]` → `TVS-CUS-[CH]-[YYMM]-[XXXX]`

### Agent Code Format
- เปลี่ยนจาก `AGT-[TYPE]-[YEAR]-[SERIAL]` → `AGT-[TYPE]-[YYMM]-[SERIAL]`

### Employee Form Enhancements
- Add modal: เพิ่ม dropdown ประเภทการจ้าง (EMP/FL/CT) + dropdown แผนก (12 ตัวเลือก) + Agent Code input
- Edit modal: เพิ่ม Agent Code input + department dropdown (แทน text input)
